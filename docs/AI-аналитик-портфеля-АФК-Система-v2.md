## 1. Что это и зачем

AFK Investment Copilot — AI-инструмент, который выступает в роли скептичного члена инвестиционного комитета. Анализирует документы сделок, ищет противоречия между документами, считает финансовые показатели портфеля и генерирует аналитические отчёты.

**Ключевая идея: AI не на стороне менеджмента. Он защищает капитал АФК — ищет то, что менеджмент мог скрыть или преуменьшить.**

**Ссылка:** https://investment-copilot-demo.vercel.app

**Доступ:** По паролю `InvestCopilotTest2026`

**Модель:** Claude Sonnet 4 (Anthropic). В целевом решении архитектура позволяет подключать любые LLM-модели — как облачные (Claude, GPT), так и on-premise (CoType от MWS AI). При работе с конфиденциальными данными будут использованы только модели, развёрнутые на инфраструктуре АФК, без передачи данных внешним провайдерам.

**Инфраструктура:** Бэкенд на Railway (Europe), фронтенд на Vercel (CDN). Публичный доступ 24/7.

| Бенчмаркинг | Сравнение EV/EBITDA с аналогами | Ограниченно: мультипликаторы аналогов предзагружены, не из интернета в реальном времени |

### Советник комитета — ключевая фича

8 быстрых действий для анализа материалов инвестиционного комитета:

1. **Pre-Analysis Brief** — автоматическая сводка при загрузке: метрики, перекрёстная проверка, red flags
2. **Противоречия** — расхождения цифр между документами со ссылками на страницы и ячейки
3. **Матрица рисков** — 10 рисков в 4 категориях с влиянием на NPV/IRR
4. **Рекомендация** — ЗА и ПРОТИВ + "вопросы, которые менеджмент не хочет услышать"
5. **Executive Summary** — 1-страничная сводка по сделке
6. **Чеклист DD** — что загружено, чего не хватает, red flags
7. **Вопросы комитета** — 12 вопросов, привязанных к НАЙДЕННЫМ противоречиям
8. **Драфт протокола** — готовый документ с участниками, решением, сроками

## 4. Как тестировать

### Вход

Откройте https://investment-copilot-demo.vercel.app и введите пароль: `InvestCopilotTest2026`

### Сценарий 1: Портфельная аналитика

- "У кого самая высокая долговая нагрузка?" — таблица компаний + bar chart + источники
- "Какая компания растёт быстрее всего?" — ранжирование по выручке с источниками
- "IPO pipeline" — кандидаты на IPO (СТЕПЬ, Биннофарм, МЕДСИ)

### Сценарий 2: Советник комитета

Нажмите **"Комитет"** — слева 6 документов демо-сделки. Рекомендуемый порядок:

1. **"Pre-Analysis Brief"** — автоматическая сводка: метрики, противоречия, red flags (10–15 сек)
2. **"Противоречия"** — расхождения: заполняемость 97% vs 95% vs 90%, CAPEX 3.2 vs 3.5 млрд
3. **"Вопросы комитета"** — 12 вопросов, привязанных к найденным противоречиям
4. **"Драфт протокола"** — готовый документ с участниками, решением, сроками

### Сценарий 3: Свои документы

- Загрузите свои PDF/DOCX/XLSX — скройте демо-документы глазиком — Pre-Analysis Brief
- Все быстрые действия будут работать по вашим документам

### На что обратить внимание

- Таблицы и графики генерируются автоматически
- PDF скачивается по кнопке под ответом
- "Анализ за N шагов" — видно как AI думает (thinking steps)

## 6. Что изменилось (v1 → v2)

### Промпты с реальной аналитикой

Полностью переписаны промпты для всех навыков — теперь на уровне профессионального инвестиционного аналитика:

- **Портфельная аналитика** — фреймворк финансового здоровья (покрытие процентов, долговая нагрузка, маржа), YoY/QoQ тренды, влияние на холдинг, макро-контекст
- **Инвестиционный анализ** — полный DCF (WACC, TV, sensitivity), comparables, сценарный анализ Bull/Base/Bear, red flags checklist (8 пунктов), структура инвест-меморандума
- **Исследование рынков** — TAM/SAM/SOM, supply-side + demand-side sizing, competitive positioning, M&A активность, lifecycle assessment
- **Бенчмаркинг** — peer selection criteria, trailing vs forward мультипликаторы, implied valuation, премия/дисконт с объяснениями
- **Советник комитета** — scorecard 0–10 по 5 осям (Strategic Fit, Financial, Risk, Management, Timing), adversarial reasoning (Claim → Counter-evidence → Confidence), DD gap analysis 40+ пунктов, жёсткая логика голосования

### Новые инструменты агента

Добавлены 4 новых аналитических инструмента для детерминированных расчётов:

| Инструмент | Что делает |
|---|---|
| sensitivity_analysis | 1D и 2D матрица NPV при изменении параметров |
| scenario_analysis | Bull/Base/Bear сценарии + expected NPV/IRR |
| comparable_valuation | Implied EV через медиану мультипликаторов пиров |
| financial_health_score | Оценка финансового здоровья A–F из базы знаний |

Также: portfolio_ranking теперь динамически читает данные из KB (вместо захардкоженных), calculate умеет считать WACC.

### Архитектурные улучшения

| Компонент | v1 | v2 |
|---|---|---|
| Маршрутизация | Fast-path + ReAct | Все запросы через ReAct-агент (убран fast-path) |
| Итерации агента | до 5 | до 8 |
| Контекст документов | 30 KB | 60 KB |
| Результат инструмента | 3K символов | 6K символов |
| История диалога | 6 сообщений / 500 сим. | 10 сообщений / 1500 сим. |
| Верификация | 1 этап | 2 этапа: фактическая точность + аналитическая полнота (8 критериев) |
| LLM температура | 0.3 | 0.15 (точнее, меньше "креатива") |
| Сложные запросы | Claude Sonnet | tier="deep" → Claude Opus для комитета |
| Кеш | In-memory dict | Персистентный SQLite (TTL 24ч) |
| Модель | CoType + Claude с переключением | Claude Sonnet 4 (основная) |

### Качественная база знаний

- **Макро-контекст:** ключевая ставка 21%, инфляция, доходности ОФЗ, прогнозы ЦБ, санкционное давление
- **Профили компаний (7):** competitive position, management assessment, key risks, catalysts, valuation context
- **Динамика секторов (7):** размер рынка, темпы роста, тренды, регуляторика, консолидация

## 7. Что НЕ работает в MVP

| Ограничение | Почему | Как решить |
|---|---|---|
| Нет интернета | LLM работает офлайн. Все данные только из загруженных документов | Фаза 1 roadmap: web search, актуальные котировки |
| Бенчмаркинг с аналогами — предзагруженные данные | Мультипликаторы peers (EV/EBITDA аналогов) захардкожены | Фаза 1 roadmap: web search |
| Часть данных не за полный год | Segezha: 9М 2025, Эталон: 1П 2025, СТЕПЬ: FY 2024 | Обновлять по мере публикации отчётов |
| Галлюцинации возможны | LLM иногда выдумывает цифры. Есть верификатор, но не 100% | Расширить reflection, добавить fact-checking инструменты |
| Демо-сделка — не реальная | Логистический хаб "Подмосковье" — синтетические документы | Загрузить реальные материалы сделки для тестирования |
| Рыночная аналитика ограничена | Обзоры рынков — из предзагруженных документов, не актуальные | Подключить интернет-поиск, API аналитических платформ. Собственное решение MWS AI (в разработке) |

Все эти ограничения — следствие того, что это MVP. Каждое из них решаемо.

## 8. Архитектура

```
Vercel (CDN)              Railway (Europe)           Anthropic API
┌──────────────┐       ┌─────────────────────┐    ┌──────────────┐
│  Next.js 16  │──API──│  FastAPI (Python)    │────│ Claude       │
│  Static HTML │ proxy │  ReAct Agent        │    │ Sonnet 4     │
│  React 19    │       │  Vector Store       │    └──────────────┘
└──────────────┘       │  SQLite Cache       │
                       │  Document Store     │
                       └─────────────────────┘
```

**Ключевые компоненты:**
- **ReAct-агент** — цикл Reason-Act-Observe (до 8 итераций, 60KB контекста) с 9 инструментами: portfolio_ranking, data_query, document_search, calculate (WACC), sensitivity_analysis, scenario_analysis, comparable_valuation, financial_health_score, web_search
- **Двухступенчатый Reflection** — Stage 1: фактическая точность (галлюцинации), Stage 2: аналитическая полнота (8 критериев)
- **Supervisor** — автоматическая классификация запроса по навыкам
- **Vector Store** — sentence-transformers для семантического поиска по документам
- **Персистентный кеш** — SQLite, TTL 24ч, ключ = hash(skill + message + doc_ids)
- **Tier system** — Claude Sonnet 4 для стандартных запросов, Claude Opus для комитета (tier="deep")

## 9. Что достижимо при доработке

Текущий MVP демонстрирует концепцию.

| Улучшение | Эффект |
|---|---|
| Подключение к интернету | Актуальные котировки, новости, ставка ЦБ в ответах |
| Реальные peer-мультипликаторы | Бенчмаркинг с live-данными Bloomberg/Capital IQ |
| Оттачивание на реальных документах АФК | Точность ответов 95%+ вместо текущих 88% |
| True streaming | Первый токен за 1–2 сек вместо 3–7 |
| Скоринг менеджмента | Оценка управленческих команд портфельных компаний |
| Генерация презентаций PPTX | Автоматическое создание слайдов из анализа |
| Интеграция с 1С/пр. | Прямой доступ к внутренним финансовым данным |
| Алерты по ковенантам | Автоматическое уведомление при нарушении условий |

### План развития

**Подключение к интернету**
- Поиск актуальных новостей, котировок, макроданных через web search
- Мониторинг новостей по портфельным компаниям

**Скорость и качество модели**
- True streaming: токены появляются в реальном времени, без буферизации (цель: первый токен за 1–2 сек)
- Параллелизация: supervisor + context + vector search одновременно (экономия 3–5 сек на каждом запросе)
- Улучшение reflection-верификатора: снижение false positives, точная проверка числовых данных
- Расширение инструментов ReAct-агента: больше детерминированных расчётов, меньше зависимости от LLM
- Тестирование альтернативных LLM-провайдеров для повышения качества русскоязычных ответов

**Оттачивание на реальных данных**
- Загрузка реальных отчётов, МСФО, кредитных договоров
- Fine-tuning промптов под форматы документов АФК
- Калибровка верификатора для снижения галлюцинаций
- A/B тестирование с аналитиками: точность ответов, полезность рекомендаций

**Авторизация и персонализация**
- SSO авторизация
- Роли: аналитик, директор, член комитета
- История сессий, избранные ответы

**Продвинутые фичи**
- Голосовой ввод/вывод (Speech-to-Text / TTS)
- Генерация презентаций (PPTX)
- Интеграция с внутренними системами
- Алерты: уведомления при нарушении ковенантов
- Multi-agent: несколько AI-агентов параллельно
